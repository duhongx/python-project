# 数据库结构同步客户端轻量改版 UI 设计稿

日期：2026-04-14
状态：已确认方向，可进入任务拆解
定位：在现有 PyQt6 桌面客户端基础上做轻量现代化升级，保留结构同步主流程，新增 PostgreSQL 集群管理能力。

## 1. 设计目标

1. 保留现有“连接配置 -> 结构加载 -> 结构比对 -> SQL 预览 -> Dry Run/执行同步 -> 历史记录”主流程。
2. 通过统一主框架改善入口分散、弹窗过多、页面层次不清的问题。
3. 新增“集群管理”一级模块，先做本地配置和只读总览，再逐步补最小动作。
4. 整体风格采用“轻量现代化”的企业工具界面，不做重视觉和重动画。
5. 尽量复用现有服务层、线程模型和本地 SQLite，避免大规模重构。

## 2. 设计原则

### 2.1 交互原则

1. 主操作常驻可见，避免把关键动作藏进弹窗。
2. 页面结构固定，减少用户切换不同模块时的认知成本。
3. 危险动作分区隔离，必须二次确认。
4. 筛选和执行分层，先让用户看清，再让用户操作。
5. 所有异步任务都要有稳定反馈，不依赖瞬时弹窗承接状态。

### 2.2 视觉原则

1. 左导航 + 顶部全局栏 + 右侧工作区，保持传统桌面工具稳定感。
2. 使用清晰分组、浅色背景层次和统一状态色，而不是高对比炫技设计。
3. 主页面优先表格、树、卡片和信息条，不引入复杂图形组件。
4. 强调信息密度和易读性平衡，避免当前布局的“功能有了，但重点不够明确”。

### 2.3 技术原则

1. `MainWindow` 只承担主框架和页面切换，不再承载全部业务编排。
2. “结构同步”作为独立页面组件抽离，最大化复用当前逻辑。
3. “集群管理”作为新增模块，从数据存储、页面和服务适配独立建设。
4. SQLite 继续作为本地持久化中心，新增集群、审计、设置表即可。

## 3. 全局信息架构

```text
数据库结构同步客户端
├─ 结构同步
│  ├─ 连接与结构
│  ├─ 结构比对
│  └─ SQL预览与执行
├─ 集群管理
│  ├─ 集群列表
│  ├─ 集群总览
│  └─ Patroni动作
├─ 历史与审计
└─ 系统设置
```

说明：

1. “结构同步”保留一级地位，避免老用户迁移成本过高。
2. “集群管理”单独成组，避免和结构同步的数据库连接配置混淆。
3. “历史记录”升级为“历史与审计”，覆盖比对、同步和集群动作。
4. “系统设置”仅保留最小本地配置，不做复杂权限和系统管理。

### 4.1 主框架说明

1. 左侧导航固定宽度，默认不折叠。
2. 顶部只保留全局动作：环境标识、刷新、退出。
3. 页面标题区统一承载面包屑和当前页面标题。
4. 所有模块共用底部状态栏，显示加载、成功、失败、同步进度等信息。
5. 页面切换时默认保留各页面状态，不重复初始化。

### 4.2 全局状态规范

1. 成功：绿色系。
2. 告警：黄色系。
3. 失败：红色系。
4. 信息提示：蓝色系。
5. 加载中：灰蓝系，配合进度条或 loading 文案。

## 5. 结构同步页设计

结构同步页是本次改版的核心页面，应在保持当前能力完整的前提下，优化信息层次。

```text
+--------------------------------------------------------------------------------------------------+
| 结构同步 / 结构比对                                                               [开始比对]      |
+--------------------------------------------------------------------------------------------------+
| 源连接 [source_pg ▼]   目标连接 [target_kb ▼]   Schema筛选 [df_/jk_]   对象关键字 [______]     |
| [刷新结构] [连接配置]  对象类型 [全部▼]  差异分类 [全部▼]  [只看可同步☐] [忽略仅目标端☐]       |
+--------------------------------------+-----------------------------------------------------------+
| 源端结构                             | 目标端结构                                                |
| DB: gotoenglish                      | DB: gotoenglish_kb                                         |
| ├─ df_user                           | ├─ df_user                                                 |
| │  ├─ users                          | │  ├─ users                                                |
| │  ├─ roles                          | │  ├─ roles                                                |
| └─ df_order                          | └─ df_order                                                |
|    ├─ orders                         |    ├─ orders                                               |
|    └─ order_items                    |    └─ order_items                                          |
+--------------------------------------+-----------------------------------------------------------+
| 差异结果                                                                     统计: 自动12 手工8 提示15 |
| ▸ df_user                                                                                           |
|   ☑ users.display_name      字段缺失        可自动同步                                              |
|   ☑ users.email             字段缺失        可自动同步                                              |
|   • users.status            默认值差异      需人工处理                                              |
| ▸ df_order                                                                                          |
|   • orders                  目标缺表        仅提示/后续可扩展                                       |
+--------------------------------------------------------------------------------------------------+
| 已选择: 2项   风险等级: Low   [生成SQL] [Dry Run] [执行同步] [导出报告] [清空选择]               |
+--------------------------------------------------------------------------------------------------+
| 状态提示：比对完成，共 35 处差异；当前过滤后显示 20 项                                             |
+--------------------------------------------------------------------------------------------------+
```

### 5.1 页面分区

1. 顶部第一行：连接和结构范围。
2. 顶部第二行：差异筛选。
3. 中部上半区：源端结构树和目标端结构树。
4. 中部下半区：差异结果树。
5. 底部动作区：SQL 和同步动作。
6. 最底部：状态提示。

### 5.2 关键交互

1. `连接配置`
   - 打开连接管理弹窗。
   - 保存后自动刷新连接下拉，并尽量保留原选择。

2. `刷新结构`
   - 并发刷新源端和目标端结构。
   - 刷新期间按钮置灰，状态栏显示加载进度。

3. `开始比对`
   - 仅当源端和目标端已选择且不为同实例时可执行。
   - 比对完成后差异区展开，动作区激活。

4. 差异筛选
   - 任一筛选项变化后仅刷新差异展示，不重新发起数据库比对。

5. `生成SQL`
   - 基于当前勾选差异生成 SQL 计划。

6. `Dry Run`
   - 生成 SQL 和报告，不执行数据库写入。

7. `执行同步`
   - 弹出确认框，展示目标连接、SQL 数量、风险等级。

8. `导出报告`
   - 导出当前比对结果与 SQL 计划。

### 5.3 设计重点

1. 保留当前双树结构，因为这与现有实现和用户习惯最一致。
2. 真正的操作中心是差异区和底部动作区，应在视觉上突出。
3. 风险等级前置，不等用户进入确认弹窗才看到。
4. 目标端结构树仅镜像展示，不允许用户直接修改勾选。

## 6. 集群管理设计

### 6.1 集群列表页

```text
+--------------------------------------------------------------------------------------------------+
| 集群管理 / 集群列表                                                                [+新增集群]    |
+--------------------------------------------------------------------------------------------------+
| 环境 [全部▼]   关键字 [____________]   [仅启用☑]   状态 [全部▼]                     [刷新]        |
+----+----------------+--------+-------------+-------------+----------+---------------------------+
| ID | 集群名称       | 环境   | Patroni     | PG连接      | 健康状态 | 操作                      |
+----+----------------+--------+-------------+-------------+----------+---------------------------+
|  1 | HIS-PROD       | PROD   | 3 endpoints | 已配置      | 健康     | [详情] [编辑] [停用]     |
|  2 | HIS-UAT        | UAT    | 3 endpoints | 已配置      | 告警     | [详情] [编辑] [停用]     |
|  3 | ETL-TEST       | TEST   | 1 endpoint  | 未配置      | 未完成   | [编辑]                   |
+----+----------------+--------+-------------+-------------+----------+---------------------------+
```

#### 设计说明

1. 列表页承担入口和维护职责，不承担复杂动作。
2. 行级操作控制在 `详情 / 编辑 / 启用停用`。
3. 状态字段以全局状态色呈现。
4. 支持环境、关键字、启用状态和健康状态筛选。

### 6.2 新增/编辑集群弹窗

字段建议：

1. 集群名称
2. 环境
3. 说明
4. Patroni 地址列表
5. PG 管理连接：host / port / database / user / password
6. etcd endpoint 列表

按钮建议：

1. `测试连接`
2. `保存`
3. `取消`

规则建议：

1. 地址列表支持逗号分隔，自动去空格。
2. 必填项未完成时不可保存。
3. `测试连接` 先展示分项结果，不直接影响保存能力。

#### 设计说明

1. 顶部卡片只保留关键五项，不扩成大屏式仪表盘。
2. 拓扑先用文本摘要，避免引入复杂图形库。
3. 节点明细表是总览页事实来源。
4. 最近操作区与后续“历史与审计”页面数据一致。

## 7. Patroni 动作页设计

```text
+--------------------------------------------------------------------------------------------------+
| 集群管理 / Patroni动作 / HIS-PROD                                                                  |
+--------------------------------------------------------------------------------------------------+
| 当前Leader: pg01                     候选节点 [pg02 ▼]                           [执行 switchover] |
| 目标节点 [pg02 ▼]                                                            [执行 reload]        |
+--------------------------------------------------------------------------------------------------+
| 操作说明                                                                                         |
| - switchover 需要二次确认                                                                          |
| - reload 为单节点动作                                                                             |
| - 所有动作均写入审计                                                                              |
+--------------------------------------------------------------------------------------------------+
| 最近执行结果                                                                                     |
| 时间                动作           节点      结果      摘要                                       |
| 2026-04-14 10:03    reload         pg02      成功      patroni reload accepted                   |
+--------------------------------------------------------------------------------------------------+
```

### 7.1 交互规则

1. `switchover`
   - 进入确认弹窗。
   - 用户输入确认词后才允许执行。
   - 成功或失败都写审计。

2. `reload`
   - 必须先选择目标节点。
   - 当前版本建议单节点执行。

3. 最近执行结果区
   - 默认展示最近若干条动作回执。
   - 可点击跳转到完整审计页。

### 7.2 风险控制

1. 危险动作独立成页，不和总览页混排。
2. 不做批量 switchover。
3. 不做无确认直接执行。

## 8. 历史与审计页设计

```text
+--------------------------------------------------------------------------------------------------+
| 历史与审计                                                                                        |
+--------------------------------------------------------------------------------------------------+
| Tabs: [比对记录] [同步记录] [集群操作审计]                                                        |
+--------------------------------------------------------------------------------------------------+
| 筛选: 时间范围 [____ - ____]  状态 [全部▼]  关键字 [__________]                    [刷新] [导出] |
+--------------------------------------------------------------------------------------------------+
| 列表区                                                                                           |
| 选中后右侧/下方展示详情：参数摘要、错误信息、执行结果                                              |
+--------------------------------------------------------------------------------------------------+
```

### 8.1 结构说明

1. `比对记录`
   - 展示比对任务。
   - 支持查看详情、重新比对。

2. `同步记录`
   - 展示同步批次。
   - 支持查看 SQL 执行结果详情。

3. `集群操作审计`
   - 展示集群动作日志。
   - 支持按动作类型、结果状态、关键字筛选。

### 8.2 设计说明

1. 历史与审计建议由弹窗升级为正式页面。
2. 三类记录共用统一筛选框架，但保留各自动作差异。
3. 详情建议在右侧或底部固定区域展示，避免频繁新弹窗。

## 10. 页面间流转关系

### 10.1 结构同步主流程

```text
进入结构同步
  -> 选择源端和目标端连接
  -> 刷新结构
  -> 开始比对
  -> 查看并筛选差异
  -> 勾选同步项
  -> 生成 SQL
  -> Dry Run 或 执行同步
  -> 查看历史记录
```

### 10.2 集群管理主流程

```text
进入集群列表
  -> 新增/编辑集群
  -> 测试连接
  -> 保存配置
  -> 进入集群总览
  -> 刷新查看状态
  -> 进入 Patroni 动作页
  -> 执行动作并写审计
  -> 在历史与审计中查看结果
```

## 11. 状态与异常反馈设计

### 11.1 加载态

1. 结构加载、比对、同步、集群刷新都必须显示状态栏文案。
2. 中长耗时任务应配合进度条。
3. 页面内容区可使用简短 loading 提示，不需要复杂骨架屏。

### 11.2 空态

1. 无连接配置时，结构同步页显示引导按钮“去配置连接”。
2. 无差异时，差异区显示“当前无差异项”。
3. 无集群配置时，集群列表页显示空态引导“新增集群”。

### 11.3 错误态

1. 页面级错误：内容区显示失败提示和重试入口。
2. 动作级错误：弹窗提示完整原因，状态栏显示简版信息。
3. 密码、连接和网络错误要避免泄露敏感信息。

## 12. 视觉细节建议

1. 左导航背景略深于内容区，当前项用浅蓝高亮。
2. 页面标题区使用浅色分隔条，和内容区形成稳定层次。
3. 卡片区采用细边框和浅底色，不使用重阴影。
4. 按钮分主次：
   - 主按钮：开始比对、生成 SQL、执行同步、保存
   - 次按钮：刷新、导出、清空、取消
5. 表格、树和卡片之间留出稳定间距，避免“控件堆满”的旧桌面感。

## 13. 与现有代码的映射建议

### 13.1 可直接复用

1. 结构读取、比对、SQL 生成和同步执行服务层。
2. 现有 `ComparisonPanel` 的差异展示和筛选能力。
3. 现有连接配置弹窗。
4. 现有 SQL 预览、执行结果、历史记录逻辑的核心部分。

### 13.2 需要重组

1. `MainWindow`：由业务大页改为导航壳。
2. 结构同步逻辑：抽为独立页面组件。
3. 历史记录弹窗：升级为正式页面或半页面工作区。

### 13.3 需要新增

1. 集群配置存储。
2. 集群列表页。
3. 集群总览页。
4. Patroni 动作页。
5. 集群操作审计。
6. 系统设置页。

## 14. 推荐交付顺序

1. 先完成主框架和结构同步页重排。
2. 再补集群配置存储和集群列表。
3. 然后做只读集群总览。
4. 之后扩展历史与审计页。
5. 最后补系统设置和最小 Patroni 动作。

## 15. 非目标

当前版本不做：

1. 慢 SQL 分析。
2. 日志采集平台化。
3. 复杂多角色权限。
4. 大屏式图形拓扑。
5. 批量危险动作。

## 16. 结论

这版 UI 改版的重点不是推翻重做，而是：

1. 把现有结构同步主链路放进更清晰的主框架。
2. 用最小新增模块承接集群管理需求。
3. 保持企业桌面工具的稳定感，同时提升页面层次和操作效率。

如果继续推进，下一步建议基于本文档直接拆成页面级开发任务和组件级改造计划。
