# 数据库结构同步桌面客户端 PRD

## 1. 文档信息

- 产品名称：数据库结构同步桌面客户端
- 产品形态：桌面客户端
- 一期范围：数据库连接配置、用户登录、本地 SQLite 数据库、PostgreSQL 到 PostgreSQL / PostgreSQL 到 KingBase 的结构比对与缺失字段同步
- 目标平台：优先 macOS，后续可扩展 Windows
- 文档状态：需求基线
- 日期：2026-04-08

## 2. 背景

当前项目已经验证了 PostgreSQL 到 KingBase 的结构读取、差异报告和部分字段同步能力，但实现分散在命令行、Web 原型和多个桌面原型文件中。后续不再维护 Web 版本，项目统一收敛为桌面客户端。

客户端需要面向实际交付使用，提供源端数据库和目标端数据库的可视化配置、连接测试、结构比对、SQL 预览和人工确认同步能力。一期支持 PostgreSQL 到 PostgreSQL、PostgreSQL 到 KingBase 两种同步方向；其他数据库类型和更复杂的结构同步能力放入二期及后续规划。客户端自身数据使用本地 SQLite 保存，包括用户、数据库连接配置、比对任务、执行记录和报告索引。

## 3. 产品目标

一期目标是交付一个可用、可控、可追溯的桌面客户端：

- 用户可以使用默认账号登录客户端。
- 用户可以在客户端内自定义配置源端数据库和目标端数据库。
- 客户端可以保存数据库配置到本地 SQLite。
- 用户可以发起结构比对，查看源端和目标端的结构差异。
- 用户可以选择缺失字段并生成同步 SQL。
- 用户必须在预览 SQL 后手动确认执行同步。
- 客户端记录比对结果、同步记录和执行日志。
- 一期同步方向限定为 PostgreSQL 到 PostgreSQL、PostgreSQL 到 KingBase。

## 4. 用户角色

### 4.1 管理员

默认管理员账号：

- 用户名：admin
- 密码：cloudhis@2123

管理员一期具备全部功能权限，包括登录、配置数据库连接、执行比对、生成 SQL、执行同步、查看历史记录。

### 4.2 普通用户

一期不提供普通用户体系。后续版本可扩展多用户、角色和权限。

## 5. 一期功能范围

### 5.0 一期数据库支持矩阵

一期支持的数据库方向：

- PostgreSQL -> PostgreSQL。
- PostgreSQL -> KingBase。

一期不支持的数据库方向：

- KingBase -> PostgreSQL。
- KingBase -> KingBase。
- MySQL、Oracle、SQL Server、SQLite 等其他数据库。

数据库类型约束：

- 源端数据库类型一期固定为 PostgreSQL。
- 目标端数据库类型一期支持 PostgreSQL 和 KingBase。
- SQL 生成必须根据目标端数据库类型选择对应方言。
- 比对逻辑应使用统一结构模型，不直接绑定某个目标数据库实现。

### 5.1 登录与初始化

客户端首次启动时，如果本地 SQLite 数据库不存在，需要自动创建数据库文件和基础表，并写入默认管理员用户。

登录要求：

- 登录窗口包含用户名、密码、登录按钮。
- 用户名和密码不能为空。
- 密码输入框需要隐藏明文。
- 登录成功后进入主窗口。
- 登录失败时提示“用户名或密码错误”。
- 一期允许使用固定默认管理员账号，但本地数据库中仍需要保存用户记录，为后续多用户扩展预留。

密码存储要求：

- 不允许在 SQLite 中保存明文密码。
- 默认密码 `cloudhis@2123` 初始化时应保存为哈希值。
- 一期可以使用 PBKDF2、bcrypt 或 Argon2；实现阶段统一选型。

### 5.2 本地 SQLite 数据库

客户端本地数据库使用 SQLite。

默认数据库文件位置：

- 开发环境：项目目录下 `data/app.db`
- 打包后：用户应用数据目录下的 `db-schema-sync/app.db`

SQLite 一期至少包含以下数据：

- 用户表：保存登录账号和密码哈希。
- 数据库连接配置表：保存源端和目标端数据库配置。
- 比对任务表：保存每次比对的基本信息。
- 比对结果表：保存对象级和字段级差异摘要。
- 同步执行记录表：保存同步批次、SQL、状态、错误信息和执行时间。

敏感信息要求：

- 数据库连接密码不能出现在日志、报告或界面明文展示中。
- 一期可先将连接密码加密或做本地受控存储；如果暂不实现系统钥匙串，需要在技术设计中明确风险和后续替代方案。

### 5.3 数据库连接配置

用户需要在客户端内配置源端数据库和目标端数据库，不再依赖手工编辑 `config.yaml`。

源端配置字段：

- 名称：用户自定义连接名称。
- 数据库类型：一期固定为 PostgreSQL。
- Host。
- Port。
- Database。
- Username。
- Password。
- Schema 过滤前缀，默认 `df_`。
- 用户/Owner 过滤前缀，默认 `df_`。

目标端配置字段：

- 名称：用户自定义连接名称。
- 数据库类型：一期支持 PostgreSQL、KingBase。
- Host。
- Port。
- Database。
- Username。
- Password。
- Schema 过滤前缀，默认 `df_`。
- 用户/Owner 过滤前缀，默认 `df_`。

连接配置能力：

- 新增配置。
- 编辑配置。
- 删除配置。
- 测试连接。
- 设置当前使用的源端连接和目标端连接。
- 保存多组连接 Profile。
- 标记默认源端连接和默认目标端连接。
- 复制已有连接配置后另存为新配置。

校验规则：

- Host、Port、Database、Username、Password 必填。
- Port 必须为 1 到 65535 的整数。
- 源端和目标端必须都测试通过后，才允许发起结构比对。
- 目标端类型必须属于 PostgreSQL 或 KingBase。
- 源端和目标端可以指向同一数据库类型，但不允许连接配置完全相同后直接执行同步，避免误操作覆盖同一实例。

### 5.4 结构浏览

主窗口展示源端和目标端结构树。

树结构层级：

- Database。
- Schema。
- Table/View。
- Field。

字段展示信息：

- 字段名。
- 数据类型。
- 长度/精度/小数位。
- 是否可空。
- 默认值摘要。
- 字段位置。

交互要求：

- 支持展开和收起节点。
- 支持刷新结构。
- 支持按 schema 或对象名过滤。
- 支持选中某个 schema/table/view 后查看差异。
- 支持按对象类型过滤 Table/View。
- 支持按差异状态过滤结果。
- 支持只展示可同步项。

### 5.5 结构比对

一期结构比对范围：

- Schema 是否存在。
- 表是否存在。
- 视图是否存在。
- 字段是否存在。
- 字段类型是否一致。
- 字段长度/精度/小数位是否一致。
- 字段可空性是否一致。
- 字段默认值是否一致。

比对范围筛选：

- 支持 schema 前缀过滤，默认 `df_`。
- 支持用户/Owner 前缀过滤，默认 `df_`。
- 支持 schema 列表选择。
- 支持表名或视图名关键字过滤。
- 支持对象类型过滤，包含 Table 和 View。

一期自动同步范围只包含：

- 目标端缺失的表字段。

一期只展示、不自动同步的范围：

- 缺失 schema。
- 缺失表。
- 缺失视图。
- 目标端多余对象。
- 字段类型差异。
- 字段可空性差异。
- 字段默认值差异。

比对结果状态：

- 一致。
- 仅源端存在。
- 仅目标端存在。
- 字段类型差异。
- 可空性差异。
- 默认值差异。
- 不支持自动同步。

比对结果分级：

- 可自动同步：目标端缺失的表字段。
- 需人工处理：字段类型差异、可空性差异、默认值差异、serial/identity/序列相关字段。
- 仅提示：缺失 schema、缺失表、缺失视图、目标端多余对象。

比对结果需要在界面中明确区分“可同步”和“仅提示”。

### 5.5.1 数据库方言适配

一期需要引入数据库方言适配层，避免 SQL 生成逻辑写死为 KingBase。

方言适配范围：

- PostgreSQL 标识符转义。
- KingBase 标识符转义。
- PostgreSQL 字段类型格式化。
- KingBase 字段类型格式化。
- 默认值兼容性判断。
- serial/identity/序列字段风险识别。

实现要求：

- SQL 生成服务根据目标端数据库类型选择 `PostgreSQLDialect` 或 `KingBaseDialect`。
- UI 层不得直接拼接 SQL。
- 比对服务输出统一差异模型，SQL 生成服务负责转换为目标端 SQL。
- 方言适配层需要可单元测试。

### 5.6 SQL 生成与预览

用户勾选可同步字段后，可以生成 SQL 预览。

SQL 生成规则：

- 只针对目标端缺失字段生成 `ALTER TABLE ... ADD COLUMN ...`。
- SQL 必须包含 schema、table、column。
- 标识符需要正确转义，避免特殊字符、大小写和中文对象名导致执行错误。
- 字段类型需要根据源端字段定义生成。
- 如果源端字段为 `NOT NULL` 且无默认值，一期默认先添加为可空字段，并在预览中提示后续人工处理。
- 如果源端字段有默认值，需要保留默认值。
- 如果字段为 serial/identity/序列相关字段，需要提示风险；一期不自动创建序列。
- 根据目标端数据库类型生成 PostgreSQL 或 KingBase 兼容 SQL。
- 生成 SQL 时需要标记风险等级。

SQL 风险等级：

- 低风险：普通可空字段新增，且无复杂默认值。
- 中风险：带默认值字段、NOT NULL 但可降级为先添加可空字段的情况。
- 高风险：serial/identity/序列相关字段、表达式默认值、目标端类型兼容性不明确的字段。

SQL 预览要求：

- 显示将执行的 SQL 列表。
- 显示 SQL 数量。
- 显示风险提示。
- 显示目标端连接名称和数据库类型。
- 显示可执行项数量、需人工处理项数量、仅提示项数量。
- 支持复制 SQL。
- 支持保存 SQL 到文件。
- 支持 Dry Run 模式，仅生成 SQL 和报告，不连接目标端执行。
- 用户确认后才允许执行。

### 5.7 同步执行

同步执行范围：

- 一期只执行“目标端缺失字段”的新增字段 SQL。

执行要求：

- 执行前要求二次确认。
- 二次确认需要展示目标端连接、数据库类型、影响字段数、SQL 数量和风险摘要。
- 执行时显示进度。
- 每条 SQL 单独记录成功或失败。
- 单条 SQL 失败后继续执行后续 SQL，但需要在结果中明确标记失败。
- 每条 SQL 执行后需要提交或回滚当前语句。
- 执行结果写入本地 SQLite。

不允许的自动操作：

- 不自动删除目标端对象。
- 不自动修改目标端已有字段类型。
- 不自动修改目标端已有字段可空性。
- 不自动修改目标端已有字段默认值。
- 不自动 `DROP VIEW CASCADE`。
- 不自动创建缺失表或缺失视图。
- 不自动同步目标端多余对象。

### 5.8 报告与历史记录

一期需要保存：

- 比对时间。
- 源端连接名称。
- 目标端连接名称。
- 目标端数据库类型。
- 差异摘要。
- 可同步字段数量。
- 需人工处理项数量。
- 仅提示项数量。
- 用户选择的字段。
- 生成的 SQL。
- 执行结果。
- 错误信息。

界面能力：

- 查看最近比对记录。
- 查看最近同步记录。
- 打开某次同步的 SQL 和执行结果。
- 通过连接名称、目标端数据库类型、执行状态和时间范围过滤历史记录。
- 支持重新打开历史比对结果，但重新执行同步前必须重新生成 SQL 并确认。

文件报告：

- 支持导出文本或 Markdown 报告。
- 报告不能包含数据库连接密码。

## 6. 非功能要求

### 6.1 安全性

- 不在日志、报告、错误提示中输出密码。
- 本地 SQLite 中不保存用户明文密码。
- 数据库同步前必须有 SQL 预览和用户确认。
- 高风险操作不进入一期自动执行范围。

### 6.2 可用性

- 数据库连接失败时给出明确错误。
- 长时间查询和同步不能阻塞界面。
- 比对和同步过程需要有进度提示。
- 结构树数据量较大时需要支持过滤或分批加载。

### 6.3 可靠性

- 同步 SQL 逐条执行并记录结果。
- 执行失败需要保留错误详情。
- 程序异常退出后，已写入的历史记录不能损坏。
- 本地 SQLite 初始化过程需要幂等。

### 6.4 可维护性

- UI 层不得直接拼接同步 SQL。
- SQL 生成逻辑必须独立在服务层。
- 数据库元数据读取逻辑必须独立在基础设施层。
- 比对逻辑必须可单元测试。
- 数据库方言逻辑必须独立封装，避免散落在 UI 或同步执行代码中。
- Web 版本不纳入客户端一期交付范围。

## 8. 一期验收标准

- 首次启动可以自动创建 SQLite 数据库和默认管理员用户。
- 使用 `admin / cloudhis@2123` 可以登录客户端。
- 可以在客户端界面中新增、编辑、删除、测试源端和目标端数据库连接。
- 目标端连接可以选择 PostgreSQL 或 KingBase。
- 使用 PostgreSQL -> PostgreSQL 配置可以完成结构比对并生成目标端 PostgreSQL SQL。
- 使用 PostgreSQL -> KingBase 配置可以完成结构比对并生成目标端 KingBase SQL。
- 源端和目标端连接测试通过后，可以发起结构比对。
- 比对结果可以展示 schema、表、视图、字段级差异。
- 比对结果可以区分“可自动同步”“需人工处理”“仅提示”。
- 只有“目标端缺失的表字段”可以被勾选为可同步项。
- 勾选字段后可以生成真实 SQL，不允许出现占位类型如 `ADD COLUMN field type`。
- SQL 预览中能看到风险提示和待执行 SQL 数量。
- SQL 预览中能看到目标端连接名称和数据库类型。
- Dry Run 可以生成 SQL 和报告但不执行同步。
- 用户二次确认后可以执行同步。
- 同步结果按 SQL 逐条记录成功或失败。
- 历史比对和同步记录可以在客户端中查看。
- 日志、报告和界面不展示数据库密码。
- 项目不再提供 Web 作为一期交付入口。

## 9. 二期规划

以下功能不纳入一期，作为二期或后续版本规划：

- 多用户和角色权限。
- 创建缺失 schema。
- 创建缺失表。
- 创建或重建缺失视图。
- 自动修改字段类型。
- 自动修改字段可空性。
- 自动修改字段默认值。
- 同步索引、主键、外键、唯一约束、触发器、函数、存储过程。
- 自动处理视图依赖。
- 自动执行 `DROP VIEW CASCADE`。
- 数据迁移。
- 定时同步。
- 远程服务端管理。

### 9.1 二期建议目标

二期建议优先围绕“更完整的结构同步”和“更强的安全控制”展开：

- 自动创建缺失 schema，但必须支持 SQL 预览和人工确认。
- 自动创建缺失表，但只生成结构，不迁移数据。
- 支持字段类型变更 SQL 生成，默认只预览，不自动执行。
- 支持字段可空性和默认值变更 SQL 生成，默认只预览，不自动执行。
- 支持主键、唯一约束、索引、注释的比对和 SQL 生成。
- 支持回滚 SQL 生成。
- 支持任务级事务策略配置。
- 支持更细粒度的同步权限。
- 支持多用户、角色和审计日志。

### 9.2 后续规划

以下能力复杂度更高，建议放在二期之后：

- 视图依赖分析和重建顺序计算。
- 自动创建或重建视图。
- 外键、触发器、函数、存储过程同步。
- 数据迁移。
- 数据一致性校验。
- 定时比对和批量同步任务。
- 远程服务端管理。
- 支持 MySQL、Oracle、SQL Server 等其他数据库。
- Web/Streamlit 或 Browser UI 形态。

## 10. 风险与约束

- PostgreSQL 与 KingBase 的类型、默认值、序列、视图定义存在兼容性差异，一期只对安全范围内的缺失字段做自动同步。
- PostgreSQL -> PostgreSQL 和 PostgreSQL -> KingBase 需要不同的目标端 SQL 方言，不能共用未经适配的 SQL 拼接逻辑。
- 大量 schema/table/field 的加载可能导致界面卡顿，需要在技术设计中使用后台线程和分批加载。
- 本地保存数据库连接密码存在安全风险，需要在设计中明确加密方案。
- 当前项目中存在多个原型入口，重构时需要先移除 Web 和废弃桌面原型，保证只有一个正式桌面入口。
